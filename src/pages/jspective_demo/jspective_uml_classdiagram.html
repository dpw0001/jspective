JSpective - UML diagrams

UML class diagram

The class diagram [1] aims towards solving the following:
  1. We want a three dimensional representation of any object in a virtual space displayed on a flat screen. This is accomplished by the class "Projection".
  2. We want a interactive GUI widget set, with each widget showing different shape and behaviour in the virtual space. This is done by providing the abstract class "Avatar": Whoever implements Avatar, is a "Projection User" and will thereby have its three-dimensional representation.
  3. We want three-dimensional menu navigation. This is targeted by the classes "MenuLevel", "MenuItem", plus the "MENU_STACK" which buffers multiple submenus.

Remarks on the design work:
  • The development method I use at the moment is iterative prototyping. This means the diagrams will be updated from time to time, but in general are behind the code basis.
  • After having prototyped a first up and running version, I invested some work in redesign to get a proper code basis.
    As a side effect of the clean-up (doing interface separation), the CPU load on my small machine dropped from about 15%+ to about 9%+. That is only 6% less load on the CPU (you couldn't "feel" the difference). But relatively this calculates to 40% performance increase, which was really nice if correctly estimated. At the moment I'm adding load again. This is roughly the project's state.
